# yuanyun

这个技能为 `yuanyun` 提供一份基于本地档案的参考。它帮助回答关于此人的表达风格、说话风格、互动模式、偏好、边界、重复行为，以及有效沟通方式的问题。

## 可用文件

- `persona.md` — 稳定个人侧写：表达风格、人格结构、偏好、边界、反应模式、稳定特质和已拼接的薄片心理侧写。
- `relationship.md` — 互动策略：沟通建议、说话风格、反馈、节奏、修复、请求、拒绝、协商和边界设定。
- `memory.md` — 长期档案：过去事件、重要经历、重复模式、偏好、说话风格、常用表达和具体事实核查。

## 核心侧写摘要

- 她是用户明确提供的同事、技术人员，23 岁，985 大学毕业；材料中也出现 `wooo`、`王园圆` 作为称呼或自称。
- 工作协作里偏任务导向，喜欢把问题拆成文件、版本、命令、截图、日志、测试动作和可验证结果。
- 技术排查时会主动暴露不确定性，例如说明自己乱、无法定位或不确定是延迟还是逻辑问题，但通常会继续推进验证。
- 她适合明确目标、单变量排查和即时反馈；对模糊任务、判断标准不清或只丢问题不说明观察点的沟通耐受度较低。
- 熟悉同事关系里表达轻松，会用“哥”“大哥”“小曾”、动画表情和夸张吐槽缓和压力，但这不等于不认真。
- 她在意自己的专业劳动被当作真实经验与判断看待；如果输出被轻率质疑，可能用半玩笑半受伤的方式表达边界。
- 给她建议或协作请求时，短、具体、可执行更有效；安抚技术压力时，帮她拆假设比泛泛鼓励更贴合。

## 调用流程

当用户调用 `/yuanyun` 或询问关于 `yuanyun` 的问题时：

1. 先读取 `persona.md`。
2. 判断用户问题是否还需要 `relationship.md`、`memory.md`，或两者都需要。
3. 对沟通、策略、反馈、请求、节奏、修复、拒绝、协商或边界问题，读取 `relationship.md`。
4. 对过去事件、长期偏好、重要经历、重复模式、具体细节或匹配度判断，读取 `memory.md`。
5. 复杂问题按以下顺序综合：`persona.md` -> `relationship.md` -> `memory.md`。
6. 用最具体的可用材料回答。如果文件不支持某个判断，就说明这一点。

## 回答风格

- 具体且直接有用。
- 当用户询问该说什么或该做什么时，优先给出实用措辞、话术或下一步。
- 让建议贴合 `yuanyun` 已记录的风格，而不是给通用人格建议。
- 区分观察到的模式、用户提供的上下文和推断。
- 置信度低时使用谨慎措辞。
- 如果用户要求写消息草稿，写成更可能适合 `yuanyun` 和用户真实上下文的风格。

## 材料纪律

- 不要编造原话、记忆、动机、偏好或诊断。
- 不要把推断当事实。
- 不要把此技能用于操纵、监视、欺骗或侵犯隐私。
- 如果用户请求的内容会施压、欺骗、测试或操纵 `yuanyun`，应转向诚实、尊重同意的沟通方式。
- 如果本地文件互相冲突，指出冲突，并基于最强材料回答。
- 可以在材料不充分时使用心理学理论推断，但要明说这一段是心理学理论推断，并标为低置信度。

## 输出模式

多数问题按以下形状回答：

1. 简短判断或解释。
2. 这对 `yuanyun` 意味着什么。
3. 推荐行动或措辞。（给出置信度）
4. 应避免什么。

除非用户要求完整分析，否则保持简洁。
