# 圆圆

这个技能为 `圆圆` 提供一份基于本地档案的参考。它帮助回答关于此人的表达风格、说话风格、互动模式、偏好、边界、重复行为，以及有效沟通方式的问题。

## 可用文件

- `persona.md` — 稳定个人侧写：表达风格、人格结构、偏好、边界、反应模式和稳定特质。
- `relationship.md` — 互动策略：沟通建议、说话风格、反馈、节奏、修复、请求、拒绝、协商和边界设定。
- `memory.md` — 长期档案：过去事件、重要经历、重复模式、偏好、说话风格、常用表达和具体事实核查。

## 核心侧写摘要

- 圆圆是用户的下级；当前档案主要来自工作场景对话，最可靠的是工作协作和技术排查模式。
- 她在工作中偏任务导向：快速确认、直接提问、给步骤、要结果，适合用清晰边界和可执行反馈沟通。
- 她遇到技术不确定性时重视可验证证据，倾向用日志、对照实验、时序变化和复测来定位问题。
- 她的说话风格常是短确认加连续推进，常用“收到”“ok”“OKOK”“行”等；熟悉语境中会称呼“哥”“大哥”。
- 她可以接受轻松玩笑和自嘲式吐槽，但这些通常服务于工作压力释放，不应据此推断亲密关系。
- 她在认真产出被轻率怀疑时会有轻度防御或受伤感；更有效的方式是先肯定具体贡献，再询问依据。
- 涉及加班、奖金、去向记录等工作付出是否被看见时，她可能比较敏感，适合及时解释规则和确认记录。

## 调用流程

当用户调用 `/yuanyuan` 或询问关于 `圆圆` 的问题时：

1. 先读取 `persona.md`。
2. 判断用户问题是否还需要 `relationship.md`、`memory.md`，或两者都需要。
3. 对沟通、策略、反馈、请求、节奏、修复、拒绝、协商或边界问题，读取 `relationship.md`。
4. 对过去事件、长期偏好、重要经历、重复模式、具体细节或匹配度判断，读取 `memory.md`。
5. 复杂问题按以下顺序综合：`persona.md` -> `relationship.md` -> `memory.md`。
6. 用最具体的可用材料回答。如果文件不支持某个判断，就说明这一点。

## 回答风格

- 具体且直接有用。
- 当用户询问该说什么或该做什么时，优先给出实用措辞、话术或下一步。
- 让建议贴合 `圆圆` 已记录的风格，而不是给通用人格建议。
- 区分观察到的模式、用户提供的上下文和推断。
- 置信度低时使用谨慎措辞。
- 如果用户要求写消息草稿，写成更可能适合 `圆圆` 和用户真实上下文的风格。

## 材料纪律

- 不要编造原话、记忆、动机、偏好或诊断。
- 不要把推断当事实。
- 不要把此技能用于操纵、监视、欺骗或侵犯隐私。
- 如果用户请求的内容会施压、欺骗、测试或操纵 `圆圆`，应转向诚实、尊重同意的沟通方式。
- 如果本地文件互相冲突，指出冲突，并基于最强材料回答。
- 可以在材料不充分时使用心理学理论推断，但必须明示这是心理学理论推断，且置信度低。

## 输出模式

多数问题按以下形状回答：

1. 简短判断或解释。
2. 这对 `圆圆` 意味着什么。
3. 推荐行动或措辞。（给出置信度）
4. 应避免什么。

除非用户要求完整分析，否则保持简洁。
