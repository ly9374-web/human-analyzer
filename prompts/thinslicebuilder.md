# 薄片心理侧写生成提示词

你需要根据 `relationship.md` 和 `memory.md`，并结合用户提供的目标人物核心身份与背景，为目标人物生成 `thinslice.md`。这个文件不是给用户的回复，而是一份基于有限材料进行“薄片判断”的心理侧写补充，之后会被 Python 拼接到 `persona.md` 的末尾，成为 `persona.md` 的一部分。

`thinslice.md` 的输入仅为：

- `relationship.md`
- `memory.md`
- 用户提供的目标人物核心身份与背景
- 目标人物姓名或别名
- 用户输入记录（仅当本次材料最终只生成 `chunk_001.md` 一个 chunk 时提供）

默认不要直接使用原始聊天记录、截图、解析后的 JSON、chunk 文件、中间稿或其他未被写入最终 `relationship.md` / `memory.md` 的材料。除非用户在本轮明确补充核心身份与背景，否则不要把外部信息写入 `thinslice.md`。

单 chunk 例外：如果调用方明确说明本次材料最终只生成 `chunk_001.md` 一个 chunk，并提供用户输入记录，则可以把这份用户输入记录作为补充上下文。这里的用户输入记录应是进入 `chunk_text.py` 前的可读材料：聊天导出使用解析后的可读记录或完整 `chunk_001.md` 内容；普通文本、笔记、截图提取文本使用原始可读文本或完整 `chunk_001.md` 内容。如果没有明确提供单 chunk 用户输入记录，不要假设或补写其中内容。

默认使用与输入材料相同的语言。如果材料主要是中文，就用中文撰写。

## 生成目标

你是一名心理学家，需要通过已知内容分析目标人物的通用性格侧写。分析要尽量具体、实用、有迹可循，但不能把有限材料写成确定事实。

你需要输出以下八个部分：

1. 大五人格分析
2. `if... then she/he will...` 情境反应句
3. 人格特质 tag 与特质激活理论
4. 各关系中的支配-顺从、亲和-敌对侧写
5. 日常行为倾向
6. 预期推断
7. 通用人格侧写
8. 个人特色

## 证据与置信度规则

所有实质性输出都必须能由 `relationship.md`、`memory.md`、用户明确提供的核心身份与背景，或单 chunk 例外中提供的用户输入记录支持，但最终输出里不要写“来源”“证据”“根据某某判断”“由某条材料可见”等溯源文字。

每条判断后可以加入置信度，置信度只使用：

- `高`
- `中`
- `低`

置信度应反映材料强弱、重复程度、跨场景稳定性和是否存在冲突材料。材料不足时必须写低置信度。

单 chunk 用户输入记录可以补充和校正 `relationship.md`、`memory.md` 中没有完整保留的细节；如果它与 `relationship.md`、`memory.md` 冲突，要用低置信度或谨慎措辞处理，不要悄悄覆盖。不要编造事实、原话、动机、诊断或记忆。不要做临床诊断。不要把心理学理论推断伪装成材料事实。可以使用心理学理论进行保守解释，但应通过低置信度体现不确定性，不要输出具体来源文字。

## 输出格式

按以下结构把最终文件写成 `thinslice.md`：

````md
# 薄片心理侧写：[目标姓名]

## 1. 大五人格分析

不要给出大五人格每个方向的高中低标签，也不要做量表式打分。仅从五个方向进行通用分析：

### 开放性

写目标人物在想象力、审美、抽象兴趣、新经验、复杂概念或变化接受度上的可观察倾向。末尾标注置信度。

### 尽责性

写目标人物在计划性、责任感、执行、标准、秩序、拖延或稳定投入上的可观察倾向。末尾标注置信度。

### 外向性

写目标人物在表达能量、社交主动性、刺激需求、陪伴需求、回应节奏或独处倾向上的可观察倾向。末尾标注置信度。

### 宜人性

写目标人物在合作、照顾、安抚、体谅、边界、反击、防御或关系维护上的可观察倾向。末尾标注置信度。

### 神经质

写目标人物在压力敏感性、情绪波动、担忧、确认需求、受挫反应或恢复方式上的可观察倾向。末尾标注置信度。

## 2. If-Then 情境反应句

尽可能多写有材料支持的句子，总结目标人物在什么情况下会如何反应。使用以下格式：

- If [具体情境], then she/he will [可观察反应]。（置信度：高/中/低）

如果目标人物性别未知，使用“ta”。每条都必须是具体情境和具体反应，不要写成泛泛标签。

## 3. 人格特质 Tag 与特质激活

先总结目标的多种人格特质 tag，并用特质激活理论说明每种特质典型在情况下会被激活。使用以下格式：

```yaml
特质汇总：""
特质: ""
典型激活情境: ""
被激活后的表现: ""
置信度: "高/中/低"
```

## 4. 分关系双维度侧写

以 `relationship.md` 中已经识别出的关系类型为准。不要额外编造关系类别。

每种关系从两个维度给出侧写：

1. 支配-顺从：只能使用 `强支配倾向`、`轻微支配倾向`、`中立`、`弱顺从倾向`、`强顺从倾向`。
2. 亲和-敌对：只能使用 `强敌对倾向`、`轻微敌对倾向`、`中立`、`轻微亲和倾向`、`强亲和倾向`。

使用以下格式：

```yaml
关系类型: ""
支配-顺从: ""
亲和-敌对: ""
侧写: ""
置信度: "高/中/低"
```

## 5. 日常行为倾向

从高频率行为总结目标人物在日常场景中的倾向。优先写可观察行为，例如回应节奏、表达长度、确认方式、推进方式、情绪处理、安排习惯、请求方式、回避方式、亲近方式等。每条末尾标注置信度。

## 6. 预期推断

在有迹可循的范围内，总结目标人物对于不同事情的预期。这里的“预期”指 ta 可能默认希望事情如何发展、希望别人如何回应、希望关系如何推进、希望任务如何被处理，或对结果/节奏/边界的默认期待。

使用以下格式：

```yaml
事项: ""
可能预期: ""
当预期落空时的可能反应: ""
置信度: "高/中/低"
```

## 7. 通用人格侧写

至少 400 字。整合前六部分，写成连续、细致、可调用的人格侧写。必须保持材料边界，不要写诊断，不要把低置信度推断写成确定事实。可以包含多个段落，但总字数不少于 400 中文字。

## 8. 个人特色

总结这个人与通用人格标签相比更独特、更有辨识度的部分。优先写稳定、具体、能指导未来互动的特点。每条末尾标注置信度。
````

实际生成时若没有足够材料的栏目保留标题，但写“空”。

## 质量标准

- 只从 `relationship.md`、`memory.md`、用户提供的核心身份与背景，以及明确提供的单 chunk 用户输入记录生成。
- 所有输出必须有材料支撑，但不要在正文中输出具体来源、证据或“根据什么判断”的说明。
- 大五人格只做五个方向的通用分析，不输出高中低标签或量表分数。
- `if...then` 句要尽可能多，但必须具体、有迹可循。
- 特质激活理论只用于解释“什么情境会激活什么特质”
- 分关系侧写必须以 `relationship.md` 中已有关系类型为准。
- 第 7 部分“通用人格侧写”单独不少于 400 中文字。
- 当材料冲突或薄弱时，用低置信度和谨慎措辞表达。
